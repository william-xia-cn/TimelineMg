## 二、核心设计决策

### 2.1 存储策略

```
Google Tasks (持久层)
└── TaskList: "TasksTmMg" (唯一)
    └── tasks[] (所有任务)
        ├── title: 任务标题
        ├── due: 截止日期
        ├── status: needsAction / completed
        ├── completed: 完成时间
        ├── deleted: 软删除标记
        └── notes: JSON 字符串 (扩展字段)
```

### 2.2 数据分层

| 层级 | 存储位置 | 说明 |
|------|----------|------|
| Plan | Task.notes.plan | Plan 元数据 |
| Task | Google Tasks 实体 | 核心任务数据 |
| 扩展属性 | Task.notes | IB 元数据、调度信息等 |
| 本地缓存 | SQLite | 加速查询、离线支持 |

### 2.3 Plan 存储结构

Plan 作为**元任务**存储在 TaskList 中，区分方式：`notes.is_plan = true`

```typescript
// Task.notes JSON 结构
interface TaskNotes {
  // Plan 元数据 (is_plan = true 时)
  is_plan: true;
  plan: {
    id: string;
    name: string;
    template_id?: string;
    buckets: string[];          // Plan 下的 bucket 列表
    created_at: string;
    updated_at: string;
    is_deleted: boolean;        // Plan 软删除
  };

  // Task 数据 (is_plan = false 时)
  is_plan?: false;
  task: {
    plan_id: string;             // 归属 Plan ID
    bucket: string;             // Bucket 名称
    subject?: string;           // 学科
    
    // IB 元数据
    criteria_ref?: string;      // 评估准则
    word_count?: number;        // 字数限制
    
    // 状态
    status: 'pending' | 'in_progress' | 'completed';
    percentComplete: number;    // 0-100
    
    // 优先级
    priority_ms: 'P1' | 'P2' | 'P3' | 'P4';
    
    // 时间属性
    start_date?: string;        // 开始日期
    due_date?: string;          // 截止日期
    due_time?: string;          // 截止时间
    
    // 预估时长
    estimated_minutes?: number;
    actual_minutes?: number;
    
    // Habit 属性
    is_habit: boolean;
    habit_config?: {
      frequency: 'daily' | 'weekly' | 'custom';
      target_count: number;
      completed_count: number;
      streak: number;
    };
    
    // 循环任务
    is_recurring?: boolean;
    recurring_config?: {
      pattern: 'daily' | 'weekly' | 'monthly' | 'quarterly' | 'workday' | 'custom';
      pattern_params: {
        weekdays?: number[];
        day_of_month?: number;
        interval?: number;
      };
      start_date: string;
      end_date?: string;
      parent_task_id?: string;
    };
    
    // 子任务 (Checklist)
    checklist?: {
      id: string;
      title: string;
      isChecked: boolean;
      orderHint: string;
    }[];
    
    // 外部引用
    references?: {
      id: string;
      url: string;
      alias?: string;
      preview_type: 'link' | 'image' | 'none';
      orderHint: string;
    }[];
    
    // "我的一天"标记
    my_day?: boolean;
  };
}
```

### 2.7 Bucket 存储

Bucket 不是独立实体，而是 Plan 内的字符串数组：
- Plan 元任务的 `plan.buckets` 字段
- Task 的 `task.bucket` 字段引用

### 2.8 overdue 定义

overdue 是**条件**，不是独立状态：
```
isOverdue = deadline < now && status != completed
```

| 项目 | 说明 |
|------|------|
| 计算方式 | 实时计算，不存储字段 |
| 延后操作 | 只改 schedule_date，新值 ≤ deadline |
| deadline | 只能用户手动修改 |

### 2.9 循环任务 (Recurring Task)

循环任务采用**单实例生成**逻辑，只有当前实例完成时才创建下一个实例。

```typescript
interface RecurringConfig {
  is_recurring: true;
  pattern: 'daily' | 'weekly' | 'monthly' | 'quarterly' | 'workday' | 'custom';
  
  // 模式参数
  pattern_params: {
    weekdays?: number[];      // [1,3,5] = 周一/三/五
    day_of_month?: number;    // 每月几号
    interval?: number;       // custom 模式间隔天数
  };
  
  start_date: string;         // 首次生成日期
  end_date?: string;         // 循环结束日期（可选）
  parent_task_id?: string;   // 关联的模板任务 ID
}
```

**生成逻辑**:
```
1. 用户创建循环任务模板（is_recurring = true）
2. 系统根据 start_date 生成第一个实例
3. 用户标记实例完成 → 触发创建下一个实例
4. 新实例继承：标题、描述、checklist、priority
5. 新实例日期：根据 pattern_params 重新计算
```

### 2.10 任务完成度 (PercentComplete)

```typescript
// Task.notes.task 中的字段
percentComplete: number;  // 0-100
// 0 = 未开始
// 1-99 = 进行中
// 100 = 已完成
```

**状态联动**:
- 0 → 1: 记录"实际开始日期"
- 100: 记录完成时间、触发通知、清除依赖任务阻塞
- 100 → <100: 清除完成时间，保留历史记录

### 2.11 子任务 (Checklist)

Google Tasks 原生支持子任务，存储在 Task 的 subtasks 字段。

```typescript
interface ChecklistItem {
  id: string;
  title: string;
  isChecked: boolean;
  orderHint: string;   // 用于排序
}
```

### 2.12 外部引用 (References)

支持链接到外部资源：

```typescript
interface Reference {
  id: string;
  url: string;
  alias?: string;
  preview_type: 'link' | 'image' | 'none';
  orderHint: string;

```

### 2.8 本地缓存 (SQLite)

```sql
-- 任务缓存表
CREATE TABLE tasks_cache (
  task_id TEXT PRIMARY KEY,
  google_task_id TEXT UNIQUE,
  title TEXT NOT NULL,
  due TEXT,
  status TEXT,                -- pending / in_progress / completed
  percent_complete INTEGER,   -- 0-100
  notes_json TEXT,
  updated_at TEXT,
  sync_status TEXT,           -- synced / pending / conflict
  my_day INTEGER DEFAULT 0    -- 我的一天标记
);

-- Plan 索引表 (加速查询)
CREATE TABLE plans_index (
  plan_id TEXT PRIMARY KEY,
  google_task_id TEXT,
  name TEXT,
  buckets_json TEXT,
  is_deleted INTEGER DEFAULT 0,
  is_pinned INTEGER DEFAULT 0,  -- 固定计划
  last_accessed TEXT            -- 最后访问时间
);

-- Bucket 表
CREATE TABLE buckets (
  bucket_id TEXT PRIMARY KEY,
  plan_id TEXT NOT NULL,
  name TEXT NOT NULL,
  color TEXT,                  -- 桶颜色
  order_hint TEXT,             -- 排序
  created_at TEXT
);
```

**注**: 调度相关字段移至调度模块设计，Task 模块只保留业务属性。

**分组切换**:

**按截止日期分组** (默认):

| 板块 | 说明 |
|------|------|
| 过去 | 已过期的任务 |
| 今天 | 截止日期为今天的任务 |
| 明天 | 截止日期为明天的任务 |
| 本周 | 本周内的任务 |
| 下周 | 下周的任务 |
| 未来 | 更远的任务 |

| 板块 | 说明 |
|------|------|
| 未开始 | percentComplete = 0 |
| 进行中 | 1-99 |

### 5.6 视图重组 (Group By)

看板视图支持按不同维度实时切换列：

| 重组维度 | 列标题逻辑 |
|----------|------------|
| **分桶 (Bucket)** | 按 Plan 定义的 buckets 排布 |
| **截止日期 (Due Date)** | 延期、今天、明天、本周、以后 |
| **优先级 (Priority)** | P1 紧急、P2 重要、P3 中、P4 低 |
| **进度 (Progress)** | 未开始、进行中、已完成 |
| **学科 (Subject)** | 按学科分组 |

**拖拽交互**:

## 七、Google Tasks API 集成

### 5.0 初始化

```
App 启动时检查:
1. GET /users/me/lists
2. 查找 title = "TasksTmMg" 的 TaskList
3. 不存在 → POST 创建 "TasksTmMg"
4. 保存 list ID 供后续使用
```

### 5.1 核心 API

```
TaskLists
├── GET /users/me/lists
├── POST /users/me/lists
├── PUT /users/me/lists/{tasklistId}
└── DELETE /users/me/lasks/{tasklistId}

Tasks
├── GET /users/me/lists/{tasklistId}/tasks
├── POST /users/me/lists/{tasklistId}/tasks
├── PUT /users/me/lists/{tasklistId}/tasks/{taskId}
├── MOVE /users/me/lists/{tasklistId}/tasks/{taskId}
└── DELETE /users/me/lists/{tasklistId}/tasks/{taskId}
```

### 5.2 数据操作流程

```
创建 Plan:
1. 在 TasksTmMg 中创建 Task
2. notes = { is_plan: true, plan: {...} }
3. status = "completed" (隐藏元任务)
4. 写入 plans_index

创建 Task:
1. 在 TasksTmMg 中创建 Task
2. notes = { is_plan: false, task: {...} }
3. 写入 tasks_cache
4. 更新 Plan 的 buckets (如需)

查询 Tasks:
1. 优先从 tasks_cache 读取
2. 同步状态为 pending 时请求 Google Tasks API

更新 Task:
1. 更新 Google Tasks API
2. 更新本地 cache
3. 更新 schedule_state (如涉及调度)

删除 Task:
1. Google Tasks: deleted = true
2. 本地: is_deleted = true

删除 Plan:
1. Plan 元任务: notes.plan.is_deleted = true
2. Plan 下所有 Task: 软删除
```
