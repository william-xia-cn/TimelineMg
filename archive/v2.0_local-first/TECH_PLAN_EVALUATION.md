## 方案对比

### 方案 A: Chrome Extension + localStorage ✅ 推荐

| 维度 | 评分 | 说明 |
|------|------|------|
| 实现复杂度 | ⭐⭐ (低) | 前端技术，无后端 |
| 用户体验 | ⭐⭐⭐⭐ (高) | 浏览器常驻，随时可用 |
| 数据安全 | ⭐⭐⭐⭐⭐ (极高) | 数据完全本地 |
| 扩展性 | ⭐⭐⭐ (中) | 受限于 Chrome 平台 |

**优点：**
- 零后端依赖，开发部署简单
- 可读取当前 Tab 标题（快速建任务）
- 浏览器常驻，操作便捷
- 数据完全本地，隐私无忧

### 方案 B: PWA

| 维度 | 评分 | 说明 |
|------|------|------|
| 实现复杂度 | ⭐⭐⭐ (中) | 前端为主，需配置 PWA |
| 扩展性 | ⭐⭐⭐⭐ (高) | 跨浏览器，未来可加同步 |

**缺点：** 需要 Web 托管，手机端体验一般

### 方案 C: Electron 桌面应用

| 维度 | 评分 | 说明 |
|------|------|------|
| 用户体验 | ⭐⭐⭐⭐⭐ (极高) | 原生桌面体验 |
| 实现复杂度 | ⭐⭐⭐⭐ (中高) | Electron 封装复杂 |

**缺点：** 体积大(>100MB)，安装门槛高，过度工程化

### 方案 D: 纯 HTML + localStorage

**缺点：** 无模块化，代码难以维护，难以扩展

## 推荐方案：Chrome Extension + localStorage

### 理由

1. **需求匹配度最高**
   - IB 学生主要在电脑上学习
   - Chrome 是最常用浏览器
   - 简化需求下不需要多设备

2. **与原设计对比**

| 原设计 (Apps Script) | 简化后 (Chrome Extension) |
|---------------------|--------------------------|
| 自动调度算法 | 手动拖拽分配 |
| Google Tasks 存储 | localStorage/IndexedDB |
| 多端同步 | 单设备使用 |
| 6 分钟执行限制 | 无限制 |
| Google 账号依赖 | 无依赖 |

### 核心功能映射

| 原模块 | Chrome Extension 实现 |
|--------|---------------------|
| Module 1 日历容器 | containers.html 容器配置 |
| Module 2 任务管理 | tasks.html 看板视图 |
| Module 3 Focus | focus.html 当下视图 |
| 调度引擎 | 手动拖拽 → localStorage |

---

## 结论

**在简化需求的前提下，Chrome Extension + localStorage 是最优方案：**

1. ✅ 完全满足手动调度、无滑动、无同步的需求
2. ✅ 实现复杂度低，无后端依赖
3. ✅ 用户体验好，浏览器常驻
4. ✅ 数据完全本地，隐私安全
5. ✅ 与原 Apps Script 方案相比，开发效率更高
