Privacy Policy for TimeWhere

Last Updated: May 19, 2026

TimeWhere is a Chrome extension for personal task, calendar, study planning, daily review, and optional cross-device data synchronization. TimeWhere is designed as a local-first tool: core task, calendar, planner, dashboard, MatrixView, ManageBac, reminder, and journal features work without signing in to a Google account.

1. Data TimeWhere Processes

TimeWhere may process data that the user enters, imports, or configures inside the extension, including:

  • Tasks, task status, due dates, start dates, priorities, plans, buckets, labels, and scheduling metadata.
  • Calendar containers, calendar events, timetable records, MatrixView import data, and Daily Journal entries.
  • ManageBac-related configuration and imported task/event metadata when the user provides a ManageBac events subscription link.
  • Extension settings, notification preferences, sync state, and local operational metadata needed to run the extension.
  • Google account authorization state when the user chooses to enable optional Google Drive data synchronization.

2. Local Storage

By default, TimeWhere stores data locally in the user's browser using IndexedDB and Chrome extension storage. The developer does not receive, host, sell, or access the user's local TimeWhere data.

3. Optional Google Drive Synchronization

If the user enables Google data synchronization, TimeWhere uses Chrome Identity and Google Drive appDataFolder to store a TimeWhere synchronization file in the user's own Google Drive account. This is used only for cross-device synchronization of TimeWhere data. TimeWhere does not use Google Calendar or Google Tasks APIs. The developer does not operate a separate server for this sync data and does not have access to the user's raw Google Drive appDataFolder contents.

4. ManageBac Subscription Links

If the user configures a ManageBac events subscription link, TimeWhere uses the link only to fetch and process the user's school event/task information for TimeWhere planning workflows. Such links may be private to the user or school system. Users should not share private ManageBac links publicly. If Google Drive synchronization is enabled, the user's saved TimeWhere configuration may be included in the user's own synchronized TimeWhere data.

5. Permissions

TimeWhere requests Chrome permissions only for its core purpose:

  • identity / identity.email: used only when the user enables optional Google Drive data sync and to help identify the selected Google account.
  • alarms: used for local task reminders, reminder checks, and daily journal scheduling.
  • notifications: used to show local task reminders and test notifications.
  • storage: used to store extension settings and local operational state.
  • tabs: used to open TimeWhere dashboard or focus pages from the popup or notification interactions.
  • unlimitedStorage: used to support local-first task, calendar, timetable, journal, and sync metadata storage.
  • Host permissions: Keystone ManageBac events paths are used only for user-configured ManageBac imports; Google Drive API hosts are used only for optional Google Drive appDataFolder synchronization.

6. Data Sharing and Sale

TimeWhere does not sell user data. TimeWhere does not transfer user data to third parties for advertising, creditworthiness, lending, or any purpose unrelated to the extension's single purpose. Data accessed through Google APIs is used only to provide user-facing TimeWhere synchronization and follows the Chrome Web Store User Data Policy, including Limited Use requirements.

7. Remote Code

TimeWhere does not use remote JavaScript or WebAssembly code. Extension code is packaged with the Chrome extension.

8. Changes to This Policy

This policy may be updated when TimeWhere's features, permissions, or data practices change. The latest version will be posted at this URL.

9. Contact
