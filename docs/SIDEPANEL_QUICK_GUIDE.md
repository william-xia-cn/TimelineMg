# TimeWhere Side Panel 快速使用手册

适用版本：当前 `0.2.3` 源码基线的 Side Panel / quick-add 功能。

> 注意：Chrome Web Store 首个 Private 测试发布包为 `0.2.1` policy-fix 包；本文面向 `0.2.2` 及后续包含 Side Panel 的测试包。

## 1. 打开侧边栏

1. 在 Chrome 工具栏点击 TimeWhere 图标。
2. TimeWhere 会在浏览器右侧打开 Side Panel。
3. 如果侧边栏没有出现，先确认扩展已启用，并重新加载扩展。

Side Panel 是当前的快速入口，用来在不离开当前网页的情况下查看今日任务、添加临时任务、整理今日总结，以及跳转到完整页面。

## 2. 认识界面

Side Panel 主要分为四块：

| 区域 | 用途 |
|---|---|
| 顶部统计 | 显示 `今日完成` 和 `今日待办` 数量 |
| 当前任务 | 显示今天当前应该处理的任务 |
| 快捷卡片 | `临时添加任务` 和 `今日总结` |
| 底部导航 | 打开 `仪表盘`、`任务`、`日历`、`设置` |

## 3. 查看和执行当前任务

`当前任务` 来自 TimeWhere 的 Daily Settle 计算结果，会根据今天的任务、开始日期、截止日期、时间容器和任务状态显示。

常用操作：

- `开始`：把任务标记为进行中。
- `暂停`：把进行中的任务恢复为未开始。
- `完成`：把任务标记为完成。
- `延后 1天 / 3天 / 7天`：把普通任务推迟到之后日期。

说明：

- ManageBac 来源任务可以开始、暂停、完成。
- ManageBac 来源任务不能用 Side Panel 直接延后，因为截止日期等来源字段由 ManageBac 决定。
- 展开任务卡后可以查看优先级、耗时、截止日期、说明、Checklist 等信息。

## 4. 临时添加任务

当今天出现临时作业或新任务时：

1. 点击 `临时添加任务`。
2. 填写任务标题。
3. 检查或修改 Plan、Bucket、开始日期、截止日期、优先级、耗时等字段。
4. 可选填写 Checklist、Label、Notes。
5. 点击 `保存任务`。

默认规则：

- 默认开始日期和截止日期是今天。
- 默认优先选择 English / 英文学科 Plan。
- 默认 Bucket 是 `作业`。
- 如果不是英语任务，可以在保存前切换到其他 Plan 和 Bucket。

周期任务：

- 选择 `每周` 或 `每月` 后，可以设置生成次数。
- 最多生成 12 个实例。

## 5. 今日总结

Side Panel 可以直接整理 Daily Journal：

1. 点击 `今日总结`。
2. 查看今天计划任务的完成/延误情况。
3. 填写：
   - 计划延误说明
   - 计划外任务说明
   - 今日总结
4. 点击 `保存草稿` 或 `提交总结`。

建议：

- 白天可以先保存草稿。
- 晚上复盘时再提交总结。
- 提交总结不会自动修改任务状态；任务完成情况来自任务本身的状态。

## 6. 底部导航

底部四个按钮用于打开完整功能页：

| 按钮 | 打开页面 | 适合做什么 |
|---|---|---|
| 仪表盘 | Focus Dashboard | 查看完整今日任务、日程、本周进度、消息流 |
| 任务 | Task Board | 批量管理任务、Plan、Bucket、筛选和分组 |
| 日历 | Calendar | 查看日程、时间容器、课程和任务分布 |
| 设置 | Settings | 管理提醒、数据同步、导入、ManageBac 配置 |

## 7. 推荐日常流程

早上：

1. 点击 TimeWhere 图标打开 Side Panel。
2. 查看 `今日待办` 和当前任务。
3. 如有新增作业，用 `临时添加任务` 快速记录。

学习中：

1. 对当前任务点击 `开始`。
2. 完成后点击 `完成`。
3. 暂时做不了的普通任务，用 `延后` 推迟。

晚上：

1. 打开 `今日总结`。
2. 检查计划任务完成情况。
3. 写下延误原因、计划外任务和今日总结。
4. 提交总结。

## 8. 常见问题

### 为什么有些任务不能延后？

如果任务来自 ManageBac，TimeWhere 会保护来源事实字段，不允许直接改动来源截止日期。你仍然可以修改本地执行状态，例如开始、暂停、完成。

### 为什么我看不到刚创建的任务？

确认任务的 `开始日期` 是否是今天或更早。如果开始日期在未来，任务不会进入今天的当前任务池。可以到 Task Board 或 Calendar 检查。

### 点击 TimeWhere 图标没有打开小弹窗？

当前基线使用 Chrome Side Panel 作为主入口，所以点击工具栏图标会打开右侧侧边栏，而不是传统 popup 小窗。

### 是否必须登录 Google？

不需要。TimeWhere 是 local-first 工具，不登录 Google 也可以使用任务、日程、Side Panel、今日总结等核心功能。Google 只用于可选的跨设备数据同步。

### Side Panel 和 Dashboard 有什么区别？

Side Panel 是快速入口，适合随手查看和轻量操作。Dashboard 是完整工作台，适合集中规划、查看日程、本周进度和更多细节。

## 9. 使用边界

- Side Panel 的临时添加适合快速记录任务，不替代 Task Board 的批量整理。
- 今日总结用于复盘，不会替你自动更改任务完成状态。
- Google 同步是可选功能，开启前应确认使用的是自己的 Google 账号。
- ManageBac 链接是私人订阅链接，不要截图或公开分享。
